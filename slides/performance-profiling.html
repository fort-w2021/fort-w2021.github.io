Profiling: Was dauert so lange?

We should forget about small efficiencies, say about 97% of the time: premature optimization is the root of all evil

— Donald Knuth (Tex, The Art of Computer Programming)

Prioritäten: Get them straight!

Code sollte:

  1. korrekt
  2. lesbar (d.h., leicht zu debuggen & erweitern)
  3. schnell

sein.

Computer werden immer schneller, R wird fortwährend optimiert.

Code wird also quasi von selbst schneller, aber nie und nimmer von selbst korrekter oder verständlicher.

Vorgehen

  1. Finde Flaschenhälse: Womit verbringt das Programm (zu)viel Zeit?
  2. Implementiere Alternative(n) für Flaschenhals
  3. Prüfe Korrektheit der Alternativen
  4. Vergleiche Laufzeiten

Womit verbringt das Programm viel Zeit?

Rprof() gibt detaillierte Analyse:

File foo.R:

foo <- function(reps = 20, n = 1e5){
    for(r in seq_len(reps)) {
        x <- rnorm(n)
        o <- order(x)
        x <- x[o]
    }
    invisible(NULL)
}

system.time(): quick & dirty

sum_i <- function(x) {
  sum <- 0
  for (i in seq_along(x)) sum <- sum + x[i]
  sum
}
sum_v <- function(x) {
  sum <- 0
  for (value in x) sum <- sum + value
  sum
}
x <- rnorm(1e4)
# all.equal(sum(x), sum_i(x));  all.equal(sum(x), sum_v(x))
reps <- 2e2
system.time(replicate(reps, sum_i(x)))
##    user  system elapsed 
##   0.079   0.000   0.080
system.time(replicate(reps, sum_v(x)))
##    user  system elapsed 
##   0.052   0.000   0.051
system.time(replicate(reps, sum(x)))["elapsed"]
## elapsed 
##   0.004

Benchmarking mit rbenchmark

library(rbenchmark)
benchmark(

  loop_vector = sum_v(x),
  built_in = sum(x),
  replications = 200, columns = c("test", "elapsed", "relative"),
